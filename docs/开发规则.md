## 1. 核心理念

### 技术原则

1. **分层模块化**：按业务领域垂直切分，每个模块独立完整
2. **单向依赖**：业务模块只能依赖共享层，不能横向依赖
3. **依赖倒置**：模块间通过抽象接口通信
4. **测试驱动**：先写测试，后写实现
5. **安全第一**：敏感信息存放在 `.env`，严禁硬编码

### 开发工作流

```
需求分析 → 架构设计 → 编码实现 → 测试验证 → 文档更新
```

## 3. 后端开发规范 (Python)

### 3.1 编码标准

- **风格**：严格遵循 **PEP 8**
- **工具**：`black` (格式化), `isort` (导入排序)
- **类型**：所有函数必须包含 **Type Hints**

### 3.2 分层职责

#### [Model] 数据层

- **`models/`**：定义 SQLAlchemy ORM 表结构，仅包含字段和关系
- **`schemas/`**：定义 Pydantic 模型 (`*Create`, `*Update`, `*Response`)，这是 API 契约
- **`repositories/`**：封装基础 CRUD 操作，接收 `db` 和 ID，返回对象
- **`services/`**：
  - 包含复杂业务逻辑（如计算、LangChain 调用、事务管理）
  - Controller 调用 Service，Service 调用 Repository
  - **禁止**：直接访问 HTTP Request/Response 对象

#### [Controller] 控制层

- **`api/`**：FastAPI 路由文件（位于各模块内）
- **职责**：参数解析、依赖注入 (`Depends`)、调用 Service、返回 HTTP 响应
- **禁止**：
  - 直接编写 SQL 语句
  - 包含复杂业务逻辑（应放入 Service）
  - 直接返回 ORM 对象（必须通过 Pydantic 序列化）

### 3.3 LangChain 规范

- **定义**：State 必须使用 `TypedDict`
- **位置**：Agent 链在 `modules/conversation/agent/chains/`，核心在 `agent/core/`
- **异常**：节点内部必须捕获异常，防止工作流挂起

### 3.4 导入顺序

```python
# 1. 标准库
import os
from datetime import datetime

# 2. 第三方库
from fastapi import Depends
from sqlalchemy.orm import Session

# 3. 本地模块
from backend.app.models.user import User
from backend.app.core.dependencies import get_current_user
```

### 3.5 命名规范

- 类名：`PascalCase`
- 函数/变量：`snake_case`
- 常量：`UPPER_SNAKE_CASE`
- 私有成员：`_leading_underscore`

## 4. 前端开发规范 (React)

### 4.1 编码标准

- **风格**：遵循 **Airbnb Style Guide**
- **工具**：`eslint`, `prettier`
- **类型**：`strict: true`，严禁使用 `any`，优先使用 `unknown`

### 4.2 分层职责

#### [Model] 数据层

- **`types/`**：定义 TypeScript 接口
- **同步**：必须与后端 `schemas/` 保持一致

#### [Controller] 控制层

- **`services/`**：封装 API 请求
- **职责**：处理 Token、错误码转换、数据格式化
- **禁止**：组件内部严禁直接写 `fetch`，必须调用 Service

#### [View] 视图层

- **`pages/`** & **`components/`**：
  - 负责 UI 渲染和事件处理
  - 通过 Controller 获取数据
  - **禁止**：在 JSX 中包含复杂业务逻辑

### 4.3 组件规范

- **函数式组件**：优先使用函数式组件 + Hooks
- **TypeScript**：所有组件必须定义 Props 类型
- **文件组织**：一个文件一个组件，组件文件与组件名称保持一致

### 4.4 状态管理规范

- **全局状态**：使用 Zustand stores
- **局部状态**：使用 React Hooks（useState, useReducer）
- **服务器状态**：优先考虑使用 React Query 或 SWR

## 5. 数据库与安全规范

### 5.1 数据库

- **ORM**：使用 **SQLAlchemy**
- **初始化**：使用 `app/db/panda.sql` 脚本
- **防注入**：严禁字符串拼接 SQL，必须使用 ORM 参数化

### 5.2 安全

- **密钥管理**：所有敏感信息（API Key, DB Password）必须存放在 `.env` 文件中
- **Git 忽略**：`.env` 文件**严禁**提交
- **认证**：所有接口（除登录外）必须通过依赖注入验证 Token

### 5.3 环境变量示例

```bash
# 数据库
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=panda

# JWT
SECRET_KEY=your-secret-key
ACCESS_TOKEN_EXPIRE_MINUTES=10080

# AI 模型
AI_TEXT_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_TEXT_KEY=sk-your-api-key
AI_TEXT_MODEL=qwen-max

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
```

## 6. AI 交互与配置

### 6.1 LangChain 集成

- **LLM 管理**：通过 `LangChainManager` 单例管理
- **AI 服务**：通过 `AIService` 统一接口调用
- **Prompts**：归属各模块管理
- **Chains**：使用 LCEL 构建链式调用

### 6.2 Agent 智能体系统

- **Agent Orchestrator**：核心编排器，协调所有组件
- **Patient Agent**：患者模拟，生成动态回复
- **State Update Engine**：基于规则更新患者状态
- **Crisis Detector**：检测危机状态

### 6.3 技能配置

配置文件：`modules/conversation/agent/config/skill_config.json`

```json
{
  "global_skill": {
    "enabled": true,
    "role_definition": "角色定义",
    "core_principles": ["核心原则"],
    "indicator_rules": {
      "mood_score": {
        "change_rules": [
          "护士表现出同理心 (+5~10)",
          "共情回应 (+3~8)"
        ]
      }
    },
    "cris_thresholds": {
      "mood_too_low": 15,
      "depression_too_high": 85
```

## Git 提交规范

### 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### type 类型

- `feat`: 新功能
- `fix`: 修复 bug
- `refactor`: 重构
- `docs`: 文档更新
- `chore`: 构建/工具相关

### 示例

```
feat(agent): 实现患者 Agent 智能体系统

- 添加 AgentOrchestrator 编排器
- 实现 StateUpdateEngine 状态更新引擎
- 添加 CrisisDetector 危机检测器
- 集成 Redis 实时状态管理
```
