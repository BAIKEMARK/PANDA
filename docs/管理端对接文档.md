## 一、系统概述

PANDA 是一个面向护理人员的围产期抑郁管理培训系统，包含：
- **课程学习**：THP 分层课程（L1-L4）
- **情景模拟**：与虚拟患者的 AI 对话练习
- **评估系统**：五维能力评估雷达图
- **学习仪表盘**：进度追踪与历史记录

管理端需要提供**课程内容管理**、**用户管理**、**数据统计**等功能。

## 三、数据库表结构

### 3.1 课程表 `courses`

```sql
CREATE TABLE `courses` (
  `id` char(36) NOT NULL COMMENT '课程ID (如 c-001)',
  `title` varchar(255) NOT NULL COMMENT '课程标题',
  `content_url` text COMMENT '课件PDF URL',
  `video_url` text COMMENT '视频URL (新增)',
  `sort_order` int DEFAULT '0' COMMENT '排序顺序',
  `level` enum('L1','L2','L3','L4') DEFAULT 'L1' COMMENT 'THP层级',
  `description` text COMMENT '课程描述',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`)
);
```

**字段说明：**
- `id`: 课程唯一标识，建议使用 `c-xxx` 格式
- `content_url`: PDF 课件路径，支持两种格式：
  - 相对路径（如 `/courses/xxx.pdf`）→ 需将 PDF 放到 `frontend/public/courses/` 目录
  - 完整 URL（如 `https://cdn.example.com/xxx.pdf`）
- `video_url`: 视频路径，格式同上
  - 相对路径（如 `/videos/xxx.mp4`）→ 需将视频放到 `frontend/public/videos/` 目录
  - 完整 URL（如 `https://cdn.example.com/xxx.mp4`）
- `level`: THP 培训层级
  - L1: 基础层（识别与筛查）
  - L2: 进阶层（心理支持）
  - L3: 高级层（专业干预）
  - L4: 专家层（督导与管理）

### 3.2 情景表 `scenarios`

```sql
CREATE TABLE `scenarios` (
  `id` char(36) NOT NULL COMMENT '场景ID (如 s-001)',
  `title` varchar(255) NOT NULL COMMENT '场景标题',
  `description` text COMMENT '场景描述',
  `system_prompt` text NOT NULL COMMENT 'AI系统提示词 (关键!)',
  `patient_background` text COMMENT '患者背景信息',
  `knowledge_tags` varchar(500) COMMENT '知识点标签 (逗号分隔)',
  `difficulty` int DEFAULT '1' COMMENT '难度等级 (1-5)',
  `time_period` varchar(50) COMMENT '时间节点 (如: 产后2周)',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`)
);
```

**字段说明：**
- `system_prompt`: **最关键的字段**，定义 AI 扮演的虚拟患者的人设、情绪、行为模式
- `patient_background`: 显示在对话界面顶部的患者背景卡片
- `difficulty`: 1-5 难度，影响场景选择和评估标准

### 3.3 用户表 `users`

```sql
CREATE TABLE `users` (
  `id` char(36) NOT NULL COMMENT '用户ID',
  `email` varchar(255) NOT NULL COMMENT '邮箱 (登录账号)',
  `password_hash` varchar(255) NOT NULL COMMENT '密码哈希 (bcrypt)',
  `name` varchar(100) NOT NULL COMMENT '姓名',
  `role` enum('student','admin','instructor') DEFAULT 'student',
  `created_at` datetime DEFAULT CURRENT_TIMESTAMP,
  `updated_at` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `email` (`email`)
);
```

**角色说明：**
- `student`: 学员，可访问课程、模拟训练、学习进度
- `instructor`: 讲师，同学员权限
- `admin`: 管理员，可访问系统管理菜单

### 3.4 菜单权限表 `menus` + `role_menu_permissions`

系统使用动态菜单，通过 `role_menu_permissions` 表控制不同角色可见的菜单项。

### 3.5 数据库迁移

项目的数据库迁移脚本位于 `backend/app/db/migrations/` 目录。

**执行迁移的方法：**

```bash
# 1. 登录 MySQL
mysql -u root -p

# 2. 选择数据库
USE panda;

# 3. 执行迁移脚本
SOURCE backend/app/db/migrations/add_video_url.sql;
```

**当前迁移文件：**

| 文件名 | 说明 |
|--------|------|
| `add_video_url.sql` | 添加课程视频URL字段 |

> ⚠️ **注意**：首次部署请先执行主 SQL 脚本 `backend/app/db/panda.sql`，然后再执行 migrations 目录下的增量迁移。

## 四、API 接口

### 4.1 基础信息

- **Base URL**: `http://localhost:8000/api`
- **认证方式**: Bearer Token（JWT）
- **Token 获取**: `POST /api/auth/login`

### 4.2 课程管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/courses/` | 获取课程列表 |
| GET | `/courses/{course_id}` | 获取单个课程 |
| POST | `/courses/` | 创建课程（需管理员权限） |
| PUT | `/courses/{course_id}` | 更新课程（需管理员权限） |
| DELETE | `/courses/{course_id}` | 删除课程（需管理员权限） |

**创建/更新课程请求体：**
```json
{
  "title": "课程标题",
  "content_url": "/courses/example.pdf",
  "video_url": "/videos/example.mp4",
  "sort_order": 1,
  "level": "L1",
  "description": "课程描述"
}
```

### 4.3 情景管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/scenarios/` | 获取情景列表 |
| GET | `/scenarios/{scenario_id}` | 获取单个情景 |
| POST | `/scenarios/` | 创建情景 |
| PUT | `/scenarios/{scenario_id}` | 更新情景 |
| DELETE | `/scenarios/{scenario_id}` | 删除情景 |

### 4.4 用户管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/users/` | 获取用户列表 |
| GET | `/users/{user_id}` | 获取单个用户 |
| POST | `/users/` | 创建用户 |
| PUT | `/users/{user_id}` | 更新用户 |
| DELETE | `/users/{user_id}` | 删除用户 |

### 4.5 数据统计接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/progress/dashboard` | 获取当前用户学习仪表盘数据 |

## 五、文件存储规范

### 5.1 静态资源目录

```
frontend/public/
├── courses/           # PDF 课件
│   ├── l1-overview.pdf
│   └── ...
├── videos/            # 课程视频
│   ├── intro.mp4
│   └── ...
└── example.pdf        # 默认示例 PDF
```

### 5.2 URL 格式

- **PDF 课件**: `/courses/xxx.pdf` 或完整 CDN URL
- **视频文件**: `/videos/xxx.mp4` 或完整 CDN URL
- **默认课件**: 如果 `content_url` 无效，前端自动回退到 `/example.pdf`

## 六、关键业务逻辑

### 6.1 课程视频显示逻辑

```
if (course.video_url 不为空) {
  显示视频播放器
}
显示 PDF 课件（如果 content_url 无效则用 example.pdf）
```

### 6.2 评估报告生成

对话结束时，系统自动调用 AI 生成评估报告，包含：
- 总分（0-100）
- 等级评定（优秀/良好/合格/不合格）
- 五维能力雷达图数据（A-E 类）
- 详细反馈和技术指导

## 七、开发建议

### 7.1 管理端功能清单

1. **课程管理**
   - 课程 CRUD
   - PDF/视频 上传（建议使用 OSS/CDN）
   - 课程排序

2. **情景管理**
   - 情景 CRUD
   - System Prompt 编辑器（关键）
   - 难度标签管理

3. **用户管理**
   - 用户 CRUD
   - 角色分配
   - 批量导入

4. **数据统计**
   - 学员学习进度汇总
   - 模拟训练统计
   - 评估报告导出

### 7.2 注意事项

1. **密码安全**：使用 bcrypt 哈希，工作因子 12
2. **ID 格式**：建议使用有意义的前缀（如 `c-001`, `s-001`, `u-xxx`）或 UUID
3. **字符集**：确保所有表使用 `utf8mb4_unicode_ci`
4. **CORS**：后端已配置允许 `http://localhost:5173`，生产环境需更新
