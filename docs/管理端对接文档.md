## 一、系统概述

PANDA (Perinatal Depression Nursing Training Assistant) 是一个面向护理人员的围产期抑郁管理智能培训系统，基于 THP (Timely Care for Perinatal Depression) 理论框架开发。

### 1.1 核心功能

| 功能模块 | 描述 |
|---------|------|
| **课程学习** | THP 分层课程（L1-L4），包含 PDF 课件和视频 |
| **情景模拟** | 与 AI 虚拟患者进行对话练习，支持动态状态跟踪 |
| **智能评估** | 基于 Agent 的五维能力评估（THP 评分标准） |
| **学习仪表盘** | 学习进度追踪、历史记录、成绩统计 |
| **权限管理** | 基于角色的菜单权限控制（RBAC） |
| **自杀倾向检测** | 智能识别患者自杀倾向并记录用户响应 |

### 1.2 用户角色

| 角色 | 代码标识 | 权限范围 |
|------|---------|---------|
| 学员 | `student` | 课程学习、情景模拟、查看个人进度 |
| 讲师 | `instructor` | 同学员权限 + 查看学员数据 |
| 管理员 | `admin` | 全部权限 + 系统管理功能 |

## 四、数据库表结构

### 4.1 用户表 `users`

```sql
CREATE TABLE `users` (
  `id` CHAR(36) PRIMARY KEY COMMENT '用户ID',
  `email` VARCHAR(255) NOT NULL UNIQUE COMMENT '邮箱',
  `password_hash` VARCHAR(255) NOT NULL COMMENT '密码哈希 (bcrypt)',
  `name` VARCHAR(100) NOT NULL COMMENT '姓名',
  `role` ENUM('student','instructor','admin') DEFAULT 'student',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_email` (`email`),
  INDEX `idx_role` (`role`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

**字段说明：**
- `role`: 用户角色
  - `student`: 学员
  - `instructor`: 讲师
  - `admin`: 管理员

### 4.2 课程表 `courses`

```sql
CREATE TABLE `courses` (
  `id` CHAR(36) PRIMARY KEY COMMENT '课程ID',
  `title` VARCHAR(255) NOT NULL COMMENT '课程标题',
  `content_url` TEXT COMMENT 'PDF课件URL',
  `video_url` TEXT COMMENT '视频URL',
  `sort_order` INT DEFAULT 0 COMMENT '排序顺序',
  `level` ENUM('L1','L2','L3','L4') DEFAULT 'L1' COMMENT 'THP层级',
  `description` TEXT COMMENT '课程描述',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_level` (`level`),
  INDEX `idx_sort` (`sort_order`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='课程表';
```

**THP 层级说明：**
- **L1 - 基础层**: 围产期抑郁识别与筛查
- **L2 - 进阶层**: 心理支持与沟通技巧
- **L3 - 高级层**: 专业干预与危机处理
- **L4 - 专家层**: 督导管理与质量控制

### 4.3 场景表 `scenarios`

```sql
CREATE TABLE `scenarios` (
  `id` CHAR(36) PRIMARY KEY COMMENT '场景ID',
  `title` VARCHAR(255) NOT NULL COMMENT '场景标题',
  `description` TEXT COMMENT '场景描述',
  `system_prompt` TEXT NOT NULL COMMENT 'AI系统提示词',
  `patient_background` TEXT COMMENT '患者背景信息',
  `knowledge_tags` VARCHAR(500) COMMENT '知识点标签',
  `difficulty` INT DEFAULT 1 COMMENT '难度等级 (1-5)',
  `time_period` VARCHAR(50) COMMENT '时间节点',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_difficulty` (`difficulty`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='情景场景表';
```

**关键字段：**
- `system_prompt`: 定义 AI 虚拟患者的角色、性格、情绪状态
- `difficulty`: 1-5 难度等级，影响评估标准

### 4.4 对话会话表 `chat_sessions`

```sql
CREATE TABLE `chat_sessions` (
  `id` CHAR(36) PRIMARY KEY COMMENT '会话ID',
  `user_id` CHAR(36) NOT NULL COMMENT '用户ID',
  `scenario_id` CHAR(36) NOT NULL COMMENT '场景ID',
  `status` ENUM('active','completed','abandoned') DEFAULT 'active',
  `start_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '开始时间',
  `end_time` DATETIME COMMENT '结束时间',
  `final_score` INT COMMENT '最终得分',
  `meta_data` JSON COMMENT '会话元数据',

  -- 自杀倾向检测字段
  `has_suicide_risk` BOOLEAN DEFAULT FALSE COMMENT '是否检测到自杀倾向',
  `suicide_risk_alerted` BOOLEAN DEFAULT FALSE COMMENT '用户是否报警',
  `suicide_risk_alert_time` DATETIME COMMENT '报警时间',
  `suicide_risk_first_detected` DATETIME COMMENT '首次检测时间',

  INDEX `idx_user` (`user_id`),
  INDEX `idx_scenario` (`scenario_id`),
  INDEX `idx_status` (`status`),
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`scenario_id`) REFERENCES `scenarios`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='对话会话表';
```

### 4.5 对话消息表 `chat_messages`

```sql
CREATE TABLE `chat_messages` (
  `id` CHAR(36) PRIMARY KEY COMMENT '消息ID',
  `session_id` CHAR(36) NOT NULL COMMENT '会话ID',
  `role` ENUM('user','assistant','system') NOT NULL COMMENT '角色',
  `content` TEXT NOT NULL COMMENT '消息内容',
  `meta_data` JSON COMMENT '消息元数据',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX `idx_session` (`session_id`),
  INDEX `idx_created` (`created_at`),
  FOREIGN KEY (`session_id`) REFERENCES `chat_sessions`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='对话消息表';
```

### 4.6 评估报告表 `evaluation_reports`

```sql
CREATE TABLE `evaluation_reports` (
  `id` CHAR(36) PRIMARY KEY COMMENT '报告ID',
  `session_id` CHAR(36) NOT NULL COMMENT '会话ID',
  `user_id` CHAR(36) NOT NULL COMMENT '用户ID',
  `total_score` INT COMMENT '总分 (0-100)',
  `level_assessment` VARCHAR(20) COMMENT '等级评定',

  -- THP 五维雷达图数据
  `radar_a_risk_identification` INT COMMENT 'A类-风险识别 (0-100)',
  `radar_b_communication` INT COMMENT 'B类-沟通支持 (0-100)',
  `radar_c_skill_application` INT COMMENT 'C类-技能应用 (0-100)',
  `radar_d_safety_management` INT COMMENT 'D类-安全管理 (0-100)',
  `radar_e_self_efficacy` INT COMMENT 'E类-自我效能 (0-100)',

  `state_analysis` JSON COMMENT '状态变化分析',
  `detailed_feedback` JSON COMMENT '详细反馈列表',
  `technical_guidance` TEXT COMMENT '技术指导',
  `meta_data` JSON COMMENT '其他元数据',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  UNIQUE KEY `uidx_session` (`session_id`),
  INDEX `idx_user` (`user_id`),
  INDEX `idx_total_score` (`total_score`),
  FOREIGN KEY (`session_id`) REFERENCES `chat_sessions`(`id`) ON DELETE CASCADE,
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='评估报告表';
```

## 六、权限系统

### 6.1 权限模型

系统采用 **RBAC (Role-Based Access Control)** 模型：

```
用户 (User) → 角色 (Role) → 菜单权限 (MenuPermission) → 可访问功能
```

### 6.2 预设菜单结构

```
PANDA 系统菜单
├── 学习中心
│   ├── 课程中心
│   └── 学习进度
├── 实训模拟
│   ├── 场景列表
│   └── 对话练习
├── 评估报告
│   └── 历史记录
└── 系统管理 (仅管理员)
    ├── 用户管理
    ├── 课程管理
    ├── 场景管理
    └── 菜单管理
```

### 6.3 权限检查流程

1. 用户登录后获取 Token
2. 前端请求 `/api/menus/user?role={role}` 获取可访问菜单
3. 前端根据菜单渲染导航栏
4. 后端在每个需要权限的接口验证 Token 和角色

## 七、Agent智能体系统

### 7.1 系统架构

PANDA 的 Agent 系统基于 LangChain 1.2.7 构建，包含以下核心组件：

| 组件 | 路径 | 功能 |
|------|------|------|
| LangChainManager | `core/ai/langchain_manager.py` | LLM 单例管理 |
| AgentOrchestrator | `modules/conversation/agent/core/agent_orchestrator.py` | Agent 编排器 |
| PatientAgentChain | `modules/conversation/agent/chains/patient_agent_chain.py` | 患者Agent链 |
| StateUpdateEngine | `modules/conversation/agent/core/state_update_engine.py` | 状态更新引擎 |
| CrisisDetector | `modules/conversation/agent/core/crisis_detector.py` | 危机检测器 |
| RedisStateManager | `core/services/redis_state_manager.py` | Redis 状态管理 |

### 7.2 患者状态指标

虚拟患者具有四个核心指标（0-100 分制）：

| 指标 | 说明 |
|------|------|
| `mood_score` | 心情指数（越高越好） |
| `satisfaction_score` | 满意度 |
| `depression_level` | 抑郁程度（越低越好） |
| `rapport_score` | 信任度 |

### 7.3 危机检测机制

当患者指标达到以下阈值时触发危机：

| 条件 | 阈值 | 响应 |
|------|------|------|
| 心情过低 | < 15 | 患者沉默，拒绝继续对话 |
| 满意度过低 | < 10 | 患者表示不理解，终止对话 |
| 抑郁过高 | > 85 | 患者情绪崩溃 |
| 信任度破裂 | < 10 | 患者表现出明显抗拒 |

### 7.4 自杀倾向检测

系统会自动检测患者是否表现出自杀倾向：

1. **检测触发**: AI 分析患者消息内容
2. **状态记录**: `has_suicide_risk` 字段标记为 TRUE
3. **用户响应**: 前端弹出报警选项
4. **评估影响**: 用户是否报警会影响 A 类-风险识别 维度的评分

## 八、评估系统

### 8.1 THP 五维评分标准

评估系统采用 THP 五维能力评估模型：

| 维度 | 代码 | 说明 | 分数范围 |
|------|------|------|---------|
| A类 | `risk_identification` | 风险识别能力 | 0-100 |
| B类 | `communication` | 沟通支持能力 | 0-100 |
| C类 | `skill_application` | THP 技能应用 | 0-100 |
| D类 | `safety_management` | 安全管理能力 | 0-100 |
| E类 | `self_efficacy` | 自我效能感 | 0-100 |

### 8.2 等级评定

| 总分范围 | 等级 |
|---------|------|
| 90-100 | 优秀 |
| 75-89 | 良好 |
| 60-74 | 合格 |
| 0-59 | 不合格 |

**注意**: 如果 D 类（安全管理）得分低于 60 分，总评直接为"不合格"（红线机制）。

## 十一、注意事项

### 11.1 安全规范

1. **密码安全**: 使用 bcrypt 哈希，工作因子 12
2. **Token 管理**: JWT Token 有效期 30 分钟
3. **SQL 注入**: 必须使用 ORM 参数化查询
4. **XSS 防护**: 前端输入必须转义
5. **CORS**: 生产环境需正确配置 CORS 白名单

### 11.2 ID 格式规范

| 类型 | 格式 | 示例 |
|------|------|------|
| 用户ID | `u-xxx` 或 UUID | `u-001` |
| 课程ID | `c-xxx` 或 UUID | `c-001` |
| 场景ID | `s-xxx` 或 UUID | `s-001` |
| 会话ID | UUID | `a1b2c3d4...` |

### 11.3 字符集规范

- 所有数据库表使用 `utf8mb4` 字符集
- 排序规则使用 `utf8mb4_unicode_ci`

### 11.4 API 响应规范

**成功响应：**
```json
{
  "code": 200,
  "message": "success",
  "data": {...}
}
```

**错误响应：**
```json
{
  "detail": "错误信息描述"
}
```

### 11.5 状态码规范

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 204 | 删除成功 |
| 400 | 请求参数错误 |
