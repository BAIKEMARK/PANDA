# v1.0 后台管理系统开发计划

## 一、开发目标

基于需求文档实现完整的后台管理系统，支持多机构、多角色、细粒度权限控制。

## 二、核心架构设计

### 2.1 数据库设计

#### 2.1.1 机构与权限表

```sql
-- 机构表
CREATE TABLE `organizations` (
  `id` CHAR(36) PRIMARY KEY,
  `name` VARCHAR(255) NOT NULL COMMENT '机构名称',
  `short_name` VARCHAR(100) COMMENT '简称',
  `logo_url` VARCHAR(500) COMMENT 'LOGO',
  `contact_name` VARCHAR(100) COMMENT '联系人',
  `contact_phone` VARCHAR(50) COMMENT '联系电话',
  `contact_email` VARCHAR(255) COMMENT '联系邮箱',
  `valid_until` DATETIME COMMENT '有效期',
  `status` ENUM('active', 'inactive') DEFAULT 'active',
  `config` JSON COMMENT '机构配置(证书/导出/语音/防作弊开关)',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX `idx_status` (`status`)
) COMMENT='机构表';

-- 角色表
CREATE TABLE `roles` (
  `id` CHAR(36) PRIMARY KEY,
  `code` VARCHAR(50) UNIQUE NOT NULL COMMENT '角色代码',
  `name` VARCHAR(100) NOT NULL COMMENT '角色名称',
  `description` TEXT COMMENT '角色说明',
  `scope` ENUM('system', 'org') DEFAULT 'org' COMMENT '作用域',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT='角色表';

-- 权限点表
CREATE TABLE `permissions` (
  `id` CHAR(36) PRIMARY KEY,
  `code` VARCHAR(100) UNIQUE NOT NULL COMMENT '权限代码',
  `name` VARCHAR(100) NOT NULL COMMENT '权限名称',
  `module` VARCHAR(50) NOT NULL COMMENT '模块',
  `action` VARCHAR(50) NOT NULL COMMENT '操作',
  `description` TEXT COMMENT '权限说明'
) COMMENT='权限点表';

-- 角色权限关联表
CREATE TABLE `role_permissions` (
  `role_id` CHAR(36) NOT NULL,
  `permission_id` CHAR(36) NOT NULL,
  PRIMARY KEY (`role_id`, `permission_id`),
  FOREIGN KEY (`role_id`) REFERENCES `roles`(`id`),
  FOREIGN KEY (`permission_id`) REFERENCES `permissions`(`id`)
) COMMENT='角色权限关联表';

-- 用户机构关联表
CREATE TABLE `user_organizations` (
  `user_id` CHAR(36) NOT NULL,
  `org_id` CHAR(36) NOT NULL,
  `role_id` CHAR(36) NOT NULL,
  `status` ENUM('active', 'inactive') DEFAULT 'active',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`user_id`, `org_id`),
  FOREIGN KEY (`user_id`) REFERENCES `users`(`id`),
  FOREIGN KEY (`org_id`) REFERENCES `organizations`(`id`),
  FOREIGN KEY (`role_id`) REFERENCES `roles`(`id`)
) COMMENT='用户机构关联表';
```

#### 2.2.2 权限中间件

```python
# backend/app/common/middleware/permission.py

from functools import wraps
from fastapi import Depends, HTTPException
from backend.app.modules.auth.services.auth_service import get_current_user

def require_permission(permission_code: str):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            user = kwargs.get('current_user')
            if not user:
                raise HTTPException(401, "未认证")
            
            # 检查用户权限
            if not has_permission(user, permission_code):
                raise HTTPException(403, "权限不足")
            
            return await func(*args, **kwargs)
        return wrapper
    return decorator
```

## 五、技术要点

### 5.1 数据隔离
- 所有查询自动过滤机构ID
- 超级管理员可跨机构查看

### 5.2 版本锁定
- 班级引用的内容自动锁定版本
- 后续更新不影响已锁定版本

### 5.3 审计追踪
- 所有关键操作记录审计日志
- 支持按时间/用户/资源类型查询

### 5.4 性能优化
- 列表查询分页
- 导出任务异步化
- 统计数据缓存
