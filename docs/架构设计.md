# 架构设计

PANDA 系统采用前后端分离的现代化架构，结合 AI 智能体技术，实现智能化的护理培训系统。

## 整体架构

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   React 19     │    │   FastAPI      │    │   通义千问 AI    │
│   前端应用      │◄──►│   Python 后端   │◄──►│   LLM 服务       │
└─────────────────┘    └────────┬────────┘    └─────────────────┘
                                │
                    ┌───────────┴───────────┐
                    ▼                       ▼
              ┌─────────┐            ┌─────────┐
              │  MySQL  │            │  Redis  │
              │  8.0    │            │  5.0+   │
              └─────────┘            └─────────┘
```

## AI 智能体架构

### 双智能体系统

```
┌─────────────────────────────────────────┐
│        Agent Orchestrator (编排器)        │
├─────────────────┬───────────────────────┤
│                 │                       │
│   Patient Agent │    Mentor Agent       │
│   (患者模拟)    │    (导师评估)        │
│                 │                       │
│ ┌─────────────┐ │ ┌─────────────────┐ │
│ │ 动态对话    │ │ │ THP 五维评估    │ │
│ │ 状态更新    │ │ │ 能力分析反馈    │ │
│ │ 危机检测    │ │ │ 报告生成        │ │
│ └─────────────┘ │ └─────────────────┘ │
└─────────────────┴───────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│         Redis 实时状态管理               │
│  ┌──────────┐      ┌──────────────┐    │
│  │ 心情指数 │      │ 抑郁程度     │    │
│  │ 满意度   │      │ 信任度       │    │
│  └──────────┘      └──────────────┘    │
└─────────────────────────────────────────┘
```

### 患者状态模型

| 指标 | 范围 | 说明 |
|------|------|------|
| mood_score | 0-100 | 心情指数 |
| satisfaction_score | 0-100 | 满意度 |
| depression_level | 0-100 | 抑郁程度 |
| rapport_score | 0-100 | 信任度 |

### THP 五维评估

- **A. 风险识别能力** - 识别围产期抑郁风险因素
- **B. 沟通支持能力** - 有效沟通与情感支持
- **C. 技能应用能力** - 干预技能的实际应用
- **D. 安全管理能力** - 危机处理与安全转介
- **E. 自我效能感** - 护理人员的自信心

## 数据流

### 对话交互流程

```
1. 用户创建会话 → 创建 Redis 状态 → 返回会话ID
                      ↓
2. 用户发送消息 → AI 分析意图 → 更新患者状态 → 生成回复
                      ↓
3. 危机检测 → 触发预警
                      ↓
4. 结束会话 → 触发评估 → 生成报告
```

### 事件驱动架构

```python
# 发布事件
event_bus.publish("chat.session_ended", {"session_id": "xxx"})

# 订阅事件
event_bus.subscribe("chat.session_ended", evaluation_handler)
```

## 核心事件

| 事件 | 触发条件 | 处理逻辑 |
|------|----------|----------|
| CHAT_SESSION_ENDED | 会话结束 | 启动评估流程 |
| EVALUATION_GENERATED | 评估完成 | 保存报告 |
| EVALUATION_FAILED | 评估失败 | 记录错误日志 |
