## 测试步骤

### 1. 数据库迁移

执行数据库迁移：
```bash
mysql -u root -p panda < backend/app/db/migrations/apply_migrations.sql
```

或使用合并的SQL文件：
```bash
cd backend/app/db/migrations
mysql -u root -p panda < apply_migrations.sql
```

### 2. 启动后端服务

```bash
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

访问API文档：http://localhost:8000/api/docs

### 3. 启动前端服务

```bash
cd frontend
npm install
npm run dev
```

访问前端：http://localhost:5173

### 4. 测试流程

#### 4.1 登录
- 使用现有管理员账号登录（如：admin@panda.com）
- 或创建新用户并分配角色

#### 4.2 测试机构管理
1. 访问 `/admin/organizations`
2. 创建新机构
3. 编辑机构信息
4. 查看机构列表

#### 4.3 测试用户管理
1. 访问 `/admin/users`
2. 创建新用户
3. 编辑用户信息
4. 分配用户到机构

#### 4.4 测试班级管理
1. 访问 `/admin/classes`
2. 创建新班级
3. 添加学员
4. 添加任务

#### 4.5 测试权限系统
- 使用不同角色账号登录
- 验证权限控制是否生效
- 验证机构数据隔离

### 5. API测试

使用Postman或curl测试API：

```bash
# 获取Token
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@panda.com","password":"your_password"}'

# 获取机构列表
curl -X GET http://localhost:8000/api/admin/organizations \
  -H "Authorization: Bearer YOUR_TOKEN"

# 创建机构
curl -X POST http://localhost:8000/api/admin/organizations \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"测试机构","short_name":"测试","status":"active"}'
```

## 注意事项

1. **数据库迁移**：执行迁移前请备份数据库
2. **权限检查**：所有API都需要JWT Token，且需要相应权限
3. **机构隔离**：非超级管理员只能访问所属机构数据
4. **审计日志**：关键操作会自动记录到audit_logs表
