## 一、高优先级任务

### 1.1 后端任务

#### 1.1.1 修复 JWT Token 硬编码问题

**问题描述**:
多处代码中存在 `user-001` 等测试 ID 硬编码，导致权限系统无法正常工作。

**影响文件**:
- `backend/app/modules/progress/api/routers.py`
- `backend/app/modules/conversation/api/routers.py`

**修复步骤**:
1. 检查所有依赖 `user_id` 的接口
2. 统一从 JWT Token 中提取用户 ID
3. 添加 Token 验证中间件
4. 编写单元测试覆盖

**验收标准**:
- [ ] 所有接口正确解析当前登录用户 ID
- [ ] 测试三个角色（student/instructor/admin）权限隔离
- [ ] 单元测试通过

---

#### 1.1.2 实现流式响应接口 (SSE)

**问题描述**:
当前对话接口采用完整响应模式，用户体验不佳。需要实现 Server-Sent Events (SSE) 或 WebSocket 流式传输。

**影响文件**:
- `backend/app/modules/conversation/api/routers.py`
- `frontend/src/services/chat.service.ts`

**实现步骤**:
1. 后端实现 SSE 端点 `/api/conversations/chat/stream`
2. 使用 `asyncio` 和 `fastapi.responses.StreamingResponse`
3. 前端使用 `EventSource` 或 `fetch` with `ReadableStream`
4. 添加打字机效果

**技术要点**:
```python
# 后端示例
from fastapi.responses import StreamingResponse
from asyncio import sleep

async def generate_response(prompt: str):
    async for chunk in llm_stream(prompt):
        yield f"data: {chunk}\n\n"

@router.post("/chat/stream")
async def chat_stream(request: ChatRequest):
    return StreamingResponse(generate_response(request.message))
```

**验收标准**:
- [ ] 流式响应端点正常工作
- [ ] 前端实时显示 AI 回复
- [ ] 支持中断和重新连接

---

**验收标准**:

- [ ] 前端正确显示错误提示
- [ ] 错误日志完整记录

#### 2.1.3 增加 API Rate Limiting

**问题描述**:
当前没有请求频率限制，存在被滥用的风险。

**实现方案**:
使用 `slowapi` 库实现基于 Redis 的速率限制。

```python
from slowapi import Limiter
from slowapi.util import get_remote_address

limiter = Limiter(key_func=get_remote_address)

@router.post("/chat")
@limiter.limit("10/minute")
async def chat_endpoint(request: Request):
    ...
```

**验收标准**:
- [ ] 所有聊天接口限流
- [ ] 超限返回友好提示
- [ ] 不同角色不同限流策略
